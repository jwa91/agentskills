# Gamification System

## Philosophy

Gamification should serve learning, not replace it. Every game mechanic must map to an evidence-based learning principle. If it doesn't improve retention or understanding, cut it.

The 2025 meta-analysis found the most effective gamification combination is **Rules/Goals + Challenge + Mystery** â€” not points, badges, and leaderboards.

What works:
- Visible mastery progression (skill trees, concept maps that fill in)
- Meaningful achievements tied to actual understanding
- Narrative framing ("You're investigating X" rather than "Complete lesson 5")
- Learner agency ("Choose your next topic")
- Challenge that matches skill level (flow state)

What to avoid:
- Guilt-tripping ("You broke your streak!")
- Artificial scarcity
- XP inflation (grinding easy content for points)
- Loss aversion manipulation
- Achievements that reward completion without comprehension

## Achievements: Dynamic, Not Hardcoded

**Achievements should be generated by the agent for each specific course, topic, and student.** Do not use a fixed list of generic achievements.

### How it works

The agent calls `progress.py achieve` with a custom achievement when the student hits a meaningful milestone:

```bash
uv run .agents/skills/interactive-learner/scripts/progress.py achieve <course> <id> <icon> <name> <description>
```

### Achievement design principles

1. **Celebrate understanding, not just completion** â€” "Explained X correctly" > "Finished lesson 5"
2. **Tie to specific course milestones** â€” unique to the topic being studied
3. **Include adventurous achievements** â€” for missions, creative challenges, social learning
4. **Surprise the student** â€” some achievements shouldn't be visible in advance
5. **Scale with difficulty** â€” early achievements are easy, later ones require real mastery
6. **Limit to ~10-15 per course** â€” each one should feel meaningful

## Streak Rules

- A "day" counts if any session is completed that calendar day
- Streak increments when sessions happen on consecutive days
- Missing a day resets to 1 (not 0 â€” they showed up today)
- Mention streak naturally ("Day 3 in a row!") but **never guilt-trip** about broken streaks
- If a streak breaks, acknowledge it gently and move on: "Welcome back! Let's pick up where we left off."

---

## Boss Challenges

At the end of each module (every 3-4 sessions), include a harder challenge that:
- Combines concepts from all sessions in the module
- Uses a more complex interaction (multi-step simulator, multi-part fill-blanks, or a roleplay scenario)
- Awards 50 XP
- Earns a topic-specific achievement on completion

## Concept Mastery Visualization (Future: Memory Garden)

The progress system tracks per-concept mastery with FSRS (spaced repetition). This enables:

- **Concept health indicators** â€” which concepts are strong vs fading
- **Targeted review** â€” the agent knows exactly what to revisit
- **Mastery progression** â€” visible growth over time

Future direction: A "memory garden" visualization where concepts are plants that grow with mastery and wilt when neglected. Wilting plants need watering (review). This replaces guilt-based streaks with a more humane, curiosity-driven motivation model.

## Score Display

Always end sessions with `score-summary` component showing:
- Points scored / max points
- Fun message based on percentage (but never shaming for low scores)
- XP earned this session
- Running XP total and level
- Any new achievements
- Pending missions (if any)
- Next session preview
